sVault (greenfield web app at /var/www/ai/sVault.rstglobal.in) is built by a team of 12 Claude Code subagents in `.claude/agents/`: `tech-lead` (master orchestrator, opus, can edit other agents' skills), `project-setup`, `db-architect`, `api-engineer`, `auth-rbac-engineer` (opus), `billing-engineer` (Razorpay+entitlements), `notifications-engineer` (renewal-alert engine — the core differentiator), `ui-ux-designer` (app + marketing site), `search-engineer` (search + RAG "Ask sVault"), `devops-engineer` (deploy + observability), `qa-test-engineer` (testing/isolation), `security-auditor` (opus, DPDP + merge gate).

Coordination: subagents can't call each other — `tech-lead` is the hub (holds Task tool) and they share contract files in `docs/` (STACK.md, TEAM.md, SCHEMA.md, API_CONTRACT.md, PERMISSIONS.md, HANDOFFS.md append-only log). `CLAUDE.md` loads team context each session.

Stack (pinned May 2026 in docs/STACK.md): FastAPI 0.136 + Pydantic v2 + Python 3.13 (uv) · Supabase Postgres + RLS + pgvector · Next.js 16 + React 19 + Tailwind v4 + shadcn/ui · Docker + GitHub Actions/GHCR.

Connected MCP: GitHub (repo github.com/rostanai/sVault), Supabase, Stitch (design project `projects/10530408406746453783`).

**Product (defined 2026-05-30):** sVault is a **Corporate Insurance Portfolio & Renewal Management System** — an internal web app for an India-based company to manage ITS OWN insurance policies (not selling insurance), replacing manual Excel + scattered email/folder soft-copies. Policy categories: Vehicle, Machinery, Plant, Factory, Employees (group health/GPA), Key-person, Stock (raw material + finished goods). Core value = multi-channel renewal alerts (WhatsApp/Telegram/SMS/email) so renewals are never missed, plus dashboard + in-app document storage. India context: INR, GST, WhatsApp Business API, SMS DLT, DPDP Act. Key docs in `docs/`: PROJECT_BRIEF, STACK, RESEARCH (verified competitors + DLT/DPDP), FEATURES (full catalog incl. AI/RAG §9A, e-signature excluded), DECISIONS (D1-D8 locked, Q1-Q7 open), PERMISSIONS, TEAM (roster + 6 domain constraints all agents inherit), HANDOFFS. Differentiators: multi-channel alert engine (60/30/15/7/1d cadence) + permission-aware RAG. India: DLT SMS, WhatsApp utility templates, DPDP Rules 2025, INR/GST. **Confirmed multi-tenant SaaS** (2026-05-30): subscription plans (Free/Starter/Professional/Enterprise) with server-side feature-gating/entitlements, **Razorpay** payments (Subscriptions + UPI Autopay + webhooks), **14-day free trial**, **Google OAuth** + email/password auth, onboarding/registration with team invitations. 11th agent `billing-engineer` owns billing+entitlements. **Org & approvals (2026-05-30):** tenant = corporate group with **parent + subsidiary companies** (records carry tenant_id + org_id; parent admins roll up, subsidiaries isolated). **High scale** SaaS. Alert channels priority = WhatsApp+Email+SMS. **Approval workflows** in scope (renewal/new policy/vendor/high-value) routed by role/permission + hierarchy, with **self-approval** where permitted, audit-logged. **In-app subscription page** (all plans + INR cost, anytime upgrade via Razorpay) is MVP. **Platform tier (2026-05-30):** **Super Admin** (SaaS platform owner) plane ABOVE all tenants — manages subscription plans/pricing, **global secrets** (Claude AI key, WhatsApp/SMS/Razorpay/email creds, encrypted), tenant management (suspend/activate/impersonate), platform analytics; separate console. **Developer API**: scoped/hashed/rate-limited/plan-gated API keys for third-party integration + signed outbound webhooks. So permission model has 2 planes: platform (Super Admin) + tenant (Admin/Manager/Owner/Viewer). FEATURES §21–22, D18–D19. Open: group vs per-subsidiary billing (Q10), approval actions+thresholds (Q11), who self-approves (Q12), plan pricing (Q6), free-tier vs paywall (Q8). **Infra (2026-05-30):** deploy on **Vercel** (GitHub auto-deploy; FastAPI as serverless functions; scheduler via pg_cron/Vercel Cron). **Supabase** project `hgopttbpoyvmlzgzyzio` (project-scoped MCP in `.mcp.json`, needs `claude /mcp` auth). Full DB schema authored in `supabase/migrations/0001-0009` (21 tables, enums, functions, triggers, indexes incl. HNSW+trgm, two-plane org-scoped RLS, realtime publication) — see `docs/SCHEMA.md`. **Marketing website** spec in `docs/MARKETING.md` (owned by ui-ux-designer). See [[prefers-freeform-questions]].
